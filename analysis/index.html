📖 4-Cluster 해석 — 실종/발견 시나리오

🔑 차성준 한 줄 정의 (가장 직관적):
HH 위험한 지역 사이에 위험한 지역  → 구역 단위 관리 필요
HL 안전한 지역 사이에 위험한 지역  → 사각지대, 특히 주의 필요 ⭐
LH 위험한 지역 사이에 안전한 지역  → 안전 섬
LL  안전한 지역 사이에 안전한 지역  → 일반 안전 지역

유형한 줄 정의실종/발견 시사점정책 함의
HH 위험한 지역 사이의 위험한 지역 치매 노인 실종 발생 多 — 광역 핫스팟 구역 단위 관리 · 면(Area) 단위 인프라 정비
HL 안전한 지역 사이의 위험한 지역 사각지대 — LL에서 실종된 노인이 이쪽으로 흘러올 가능성, 발견 거점 특히 주의 필요 · 점(Point) 단위 즉각 조치 (스마트 횡단보도, 비상벨)
LH 위험한 지역 사이의 안전한 지역 HH 클러스터에 둘러싸인 안전 섬 — 대피 경로 후보 대피 동선·안내 표지로 활용
LL 안전한 지역 사이의 안전한 지역 일반 안전 지역. 단, 여기서 실종이 발생하면 인접 HL로 이동·발견 가능성 저우선 (모니터링만)

핵심 가설 (차성준 제안):
① HH 점 밀도 높은 지역 = 실종 발생 多. 광역 위험구역으로 우선 정비.
② LL에서 실종이 발생하면 인접한 HL 격자에서 발견 확률 높음. HL을 "사각지대 + 발견 거점" 이중 의미로 해석 가능.
③ HL 격자 1,163개는 핀셋 정책으로 최대 효과 (작은 비용으로 다수 LL 격자 커버).

🗒️ 분석 방법론 노트 — 반드시 인지하고 갈 것

• Moran's I는 PCA 기반 risk_score를 입력으로 사용. 따라서 risk_score 자체의 변수별 인사이트보다, Moran 결과(HH/HL/LH/LL) × HDBSCAN 클러스터 결합 인사이트가 핵심.
• HDBSCAN cluster = -1 인 격자(28,566개)는 어떤 군집에도 속하지 않은 외부 = 배경으로 처리. 시각화·통계에서 회색 또는 제외.
• 유효 군집은 약 37개 (이전 90~100개 추정 → 실 데이터 검증 결과 37개로 확정).
